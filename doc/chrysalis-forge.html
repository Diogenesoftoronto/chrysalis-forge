Chrysalis Forge: Evolvable Racket AgentsðŸ”—

Diogenes

 (require chrysalis-forge) package: chrysalis-forge

Chrysalis Forge is a framework for building, running, and optimizing autonomous agents in Racket. It provides:

  • DSPy-inspired optimization for prompt engineering

  • Tiered security sandboxing for safe code execution

  • Multi-user service mode with BYOK (Bring Your Own Key) support

  • ACP protocol support for IDE integration

  • Client mode for connecting to remote agents

1 InstallationðŸ”—

1.1 From SourceðŸ”—

git clone https://github.com/diogenesoft/chrysalis-forge

cd chrysalis-forge

raco pkg install

This installs two CLI commands:

  • agentd — Full agent with all tools and modes

  • chrysalis-client — Lightweight client for remote services

1.2 ConfigurationðŸ”—

Create a ".env" file in the project root or set environment variables:

OPENAI_API_KEY=sk-...

CHRYSALIS_SECRET_KEY=your-secret-for-jwt

See ".env.example" for all available options.

2 Quick StartðŸ”—

2.1 Interactive ModeðŸ”—

Start an interactive session:

agentd -i

Or run a single task:

agentd "Create a function that calculates fibonacci numbers"

2.2 Service ModeðŸ”—

Run as an HTTP service:

agentd --serve --serve-port 8080

Then connect with the client:

chrysalis-client --url http://localhost:8080

2.3 ACP ModeðŸ”—

For IDE integration (Amp Code, Zed, etc.):

agentd --acp

3 Command-Line InterfaceðŸ”—

3.1 agentdðŸ”—

The main agent CLI with full capabilities.

Usage: agentd [options] [task]

 

Options:

  --acp              Run ACP Server for IDE integration

  --serve            Run HTTP service for multi-user access

  --client           Connect to a running Chrysalis service

  -i, --interactive  Enter interactive REPL mode

  -m, --model        Set LLM model (e.g., gpt-5.2, claude-3-opus)

  -p, --priority     Set runtime priority (fast, cheap, best)

  --perms            Security level (0, 1, 2, 3, god)

  -d, --debug        Debug level (0, 1, 2, verbose)

3.2 chrysalis-clientðŸ”—

Lightweight client for connecting to remote Chrysalis services.

Usage: chrysalis-client [options]

 

Options:

  -u, --url URL      Service URL (default: http://127.0.0.1:8080)

  -k, --api-key KEY  API key or JWT token for authentication

  -h, --help         Show help

4 Core ConceptsðŸ”—

4.1 SignaturesðŸ”—

Signatures define the typed interface for LLM tasks, similar to function signatures in typed languages.

struct

(struct Signature (name ins outs)
    #:extra-constructor-name make-Signature)
  name : symbol?
  ins : list?
  outs : list?
A typed interface for an LLM task.

ins — Input fields the model receives. outs — Output fields the model should produce.

4.2 ModulesðŸ”—

Modules wrap signatures with execution logic.

procedure

(Predict sig) → any/c

  sig : any/c
Creates a basic prediction module for the given signature.

4.3 OptimizationðŸ”—

The compiler optimizes module instructions and few-shot examples.

procedure

(compile! m ctx trainset) → any/c

  m : any/c
  ctx : any/c
  trainset : list?
Optimizes the module m using instruction mutation and few-shot bootstrapping.

5 Security & SandboxingðŸ”—

Code execution uses tiered sandboxes based on the –perms level:

Level

  

Name

  

Capabilities

0

  

Read-only

  

No execution, read only

1

  

Sandbox

  

Safe Racket subset, no I/O

2

  

Limited I/O

  

File read, limited network

3

  

Full

  

Full Racket, user approval required

god

  

Unrestricted

  

Auto-approve all operations

procedure

(run-tiered-code! code level) → string?

  code : string?
  level : integer?
Executes Racket code with the specified security level.

5.1 LLM Security JudgeðŸ”—

Enable an LLM-based security reviewer for sensitive operations:

export LLM_JUDGE=true

export LLM_JUDGE_MODEL=gpt-5.2

The judge reviews file writes, shell commands, and other potentially dangerous operations.

6 Service ModeðŸ”—

Run Chrysalis as a multi-user HTTP service.

Users can provide their own LLM API keys:

curl -X POST http://localhost:8080/provider-keys \

  -H "Authorization: Bearer $TOKEN" \

  -d '{"provider": "openai", "api_key": "sk-..."}'
